WebNN Text Generation – Performance Indicators Reference

1. Per-Request Generation Metrics

These metrics are displayed as badges below each assistant message during and after text generation.

Indicator Full Name Unit Description Formula
TTFT Time to First Token ms Measures the time from when a request is submitted to when the very first output token appears. A low TTFT (ideally under 500 ms) is crucial for a responsive feel, as it minimizes the initial perceived delay. This covers the prefill phase — processing the entire input prompt through the model. Tfirst_token − Trequest_start
TPS Tokens Per Second
(Decode Throughput)		 tok/s The rate at which output tokens are produced during the decode phase only (excluding TTFT). A higher value means faster, smoother streaming. The first token is excluded from both the count and the time window because its latency is dominated by the prefill operation. (Ntokens − 1) / (Tnow − Tfirst_token) × 1000
Decode Time Decode Latency ms or s The total time spent in the autoregressive decode phase, excluding the prefill (prompt processing) phase. This isolates the pure token generation time and is the standard industry term used by vLLM, TensorRT-LLM, and other serving benchmarks. E2E − TTFT
Throughput End-to-End Throughput tok/s The overall token generation rate including both the prefill and decode phases. Represents the true system throughput as experienced by the user. This is the standard industry definition of "throughput" (as used in vLLM, TGI, llmperf benchmarks). Ntokens / (E2E / 1000)
TPOT Time Per Output Token
— Inter-Token Latency (ITL)		 ms The average time to generate each subsequent token after the first one. This determines the smoothness and speed of the streaming response. It is the mathematical inverse of TPS: TPOT = 1000 / TPS. (E2E − TTFT) / (Ntokens − 1)
E2E End-to-End Latency
— Total Latency		 ms or s The total wall-clock time from sending the request to receiving the final token. This is the time the user waits for the complete answer. Tcompletion − Trequest_start

2. Model Loading Metric

Displayed in the sidebar Configuration panel after model initialization.

Indicator Full Name Unit Description Formula
Compilation Time Model Warm-up / Compile Time ms The time taken to warm up (compile/optimize) the model after loading weights. This includes running a single-token inference to trigger any JIT compilation by the WebGPU/WebNN backend. Only measured once per model load. Tend_warmup − Tstart_warmup

3. Key Relationships

Relationship Explanation
TPOT = 1000 / TPS TPOT and TPS are mathematical inverses (both measure decode-phase speed from different perspectives).
E2E = TTFT + Decode Time Total latency is the sum of the prefill time (TTFT) and the decode time.
Decode Time = TPOT × (N − 1) Decode time equals the per-token latency multiplied by the number of decoded tokens.
Throughput < TPS E2E throughput is always lower than decode throughput (TPS) because it includes TTFT in the denominator.

4. Variable Glossary

Symbol Meaning
Ntokens Total number of output tokens generated (including the first token).
Trequest_start Timestamp when generation begins (performance.now() at start).
Tfirst_token Timestamp when the first output token is produced.
Tcompletion Timestamp when the final output token is produced.
Note: The "× 1000" factor in the TPS formula converts from milliseconds (as returned by performance.now()) to seconds for the tok/s unit.
